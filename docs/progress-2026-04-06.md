# PayFi Box Progress Snapshot (2026-04-06)

## Current delivered state

### Backend
- Added platform balance domain and APIs:
  - `POST /api/balance/deposits`
  - `POST /api/balance/deposits/{id}/start-stripe-payment`
  - `POST /api/balance/deposits/{id}/sync-stripe-payment`
  - `GET /api/balance/deposits/{id}`
  - `GET /api/balance/accounts/{user_id}`
  - `GET /api/balance/accounts/{user_id}/ledger`
  - `POST /api/balance/payments/preview`
  - `POST /api/balance/payments/confirm`
- Added balance persistence model set:
  - `platform_balance_accounts`
  - `platform_balance_ledger_entries`
  - `platform_balance_locks`
  - `fiat_deposit_orders`
- Extended payment model with funding metadata:
  - `funding_source`
  - `funding_reference_id`
- KYC safety aligned for balance deposit confirmation paths:
  - webhook-driven path checks KYC
  - `sync-stripe-payment` path checks KYC
- Added duplicate-event audit visibility for balance deposit Stripe webhooks:
  - duplicate deposit webhook deliveries now leave explicit audit markers
  - deposit metadata keeps recent duplicate webhook event history

### MCP (first usable version)
- Mounted MCP endpoint at `/mcp` in API app.

  - `get_balance_ledger`

- Added user-ownership checks for MCP deposit detail / checkout / sync actions.
- Added structured MCP error mapping (no raw ToolError leak for typical validation/user-not-found cases).

- `/mcp/` is not a browser page. It is a protocol endpoint and responds according to MCP streamable HTTP expectations.
- Frontend local startup has been intermittently unstable with background shell process management on this machine.
  - Reliable fallback: run `npm --workspace apps/web run start` in a dedicated terminal tab.
